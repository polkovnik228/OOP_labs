# ЛР-1 — Класс и инкапсуляция (Python 3.x)

## Вариант 2. Библиотека / Книги

## Цель работы

* Освоить объявление пользовательских классов.
* Разобраться с инкапсуляцией (атрибуты экземпляра, закрытые поля).
* Реализовать свойства (`@property`).
* Переопределить магические методы (`__str__`, `__repr__`, `__eq__`).
* Осознать разницу между атрибутами класса и экземпляра.

## Атрибуты класса

- `library_name` — название библиотеки  
- `current_year` — текущий год (для валидации)

## Закрытые поля

- `_title` — название книги  
- `_author` — автор  
- `_year` — год издания  
- `_pages` — количество страниц  
- `_price` — стоимость книги  
- `_is_available` — доступность книги (выдана / доступна)

## Свойства (@property)

- Только чтение:
  - `title` — название книги  
  - `author` — автор  
  - `year` — год издания  
  - `pages` — количество страниц  
  - `is_available` — доступность  

- Чтение и запись:
  - `price` — цена книги (с валидацией)

## Магические методы

- `__str__` — удобный вывод информации о книге  
- `__repr__` — техническое представление объекта  
- `__eq__` — сравнение книг (по названию, автору и году)

---

## Бизнес-методы

- `take()` — взять книгу (если доступна)  
- `return_book()` — вернуть книгу  
- `discount(percent)` — применить скидку  
- `is_big()` — определить, является ли книга большой (по количеству страниц)

## Валидация

Валидация вынесена в отдельные методы:

- `_validate_title`
- `_validate_author`
- `_validate_year`
- `_validate_pages`
- `_validate_price`

Проверяется:

- тип данных  
- пустые строки  
- диапазоны значений  
- логическая корректность  

## Логическое состояние

Книга может находиться в двух состояниях:

- доступна  
- выдана  

Ограничения:

- нельзя выдать уже выданную книгу  
- нельзя изменить цену выданной книги  
- нельзя применить скидку к выданной книге  

## Демонстрация (demo.py)

В демонстрационном файле реализовано:

- создание объектов

- демонстрация состояний
